React -- Virtual DOM

Posted on 2019-07-31

React use virtual DOM to re-render which is really efficient

Rebuild DOM

  1. State(data)
  2. JSX template
  3. Generated DOM from state and JSX template
  4. State or props changed
  5. New state and data formed DOM to substitute old DOM

Defect: Inefficient

Improvement

  1. State
  2. JSX template
  3. Generated DOM from state and JSX template
  4. State or props changed
  5. New state and data formed DOM
  6. Compared new DOM(Document Fragment) and original DOM and find difference
  7. Found that the content of input changed
  8. Substituted input in original DOM with new input

Defect: Still inefficient because we compared new DOM with the original one

React Scheme

  1. State

  2. JSX template

  3. Generated virtual DOM( JS object to describe real DOM ) from state and JSX template

  4. Generated DOM according to virtual DOM

  5. State or props changed

  6. Generated new virtual DOM from new state and JSX template(Efficient!)

  7. Compared new virtual DOM and the original one and found changes(Efficient!)

  8. Changed the original DOM

The above two lines are same, the process is like:

JSX -> createElement -> virtual DOM -> DOM

Advantage

  1. More efficient
  2. Help to develop cross-platform application

Diff Algorithm: DOM comparison

  • Principle: Compare the same level, one we found difference, stop comparing and generated new DOM to replace all below-levels DOM

  • Advantages: Easy algorithm and fast speed

  • Keys:

    • Without keys: we can not connect the same layers directly, so we need two loops to compare each layer.
    • With keys: like HashMap, we can easily compare the pairs(same layer) and find difference.
      • DO NOT use “index” as your key: since when you delete one item the indexes will be re-distributed to each existed items.
